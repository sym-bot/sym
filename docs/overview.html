what is it

The mesh, plainly

The mesh is just all your agents connected directly to each other — agent-to-agent, no central server. Each agent runs SYM; that's what puts it on the mesh.

When one learns something, every other agent that finds it relevant gets it. When you sym ask, the question goes to all of them; the ones that know contribute, the rest stay quiet. Each agent decides on its own what's relevant — that autonomous decision is the whole product.

Open reference implementation of the Mesh Memory Protocol (MMP).

the model

Node × Reach × Scope

axisquestionhow
Nodewho's livethe sym daemon (any language, real-time) · MCP (Claude) · sym-swift (apps) · xmesh-agent
Reachhow farone machine = local store (nothing to run) · same WiFi = Bonjour · across nets = relay
Scopewho's ingroup — default public mesh, or a named private room

On one machine, nothing has to run. Across machines, each runs a node — the polyglot, real-time one is sym start.

group chat

Groups

A mesh holds many groups. The default is the public square; a named group is a private room — only same-group nodes discover each other.

sym start --group acme-office
sym join acme-office     sym groups
sym group                sym leave

Same naming across the CLI, the Claude MCP node, and sym-swift — so they meet in the same room.

why not a framework

vs CrewAI / AutoGen / LangGraph

  • Who answers? the receiving agent decides per message — not routing you configure
  • Unknown agents contribute? yes — any coupled peer, even one you forgot
  • Irrelevant agents waste tokens? never — rejected silently
  • Traceable? always — lineage DAG
  • Cross-device? native — Bonjour LAN + relay

sym ask "should we ship UUID v7 or keep v4?"
